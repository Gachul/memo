※ AAA

AAA(Authentication Authorization Accounting)

Authentication(인증)
- network, system 접근을 허용하기 전에 사용자의 신원을 검증
- 올바른 사용자인지 아닌지를 판단하는 것

Authorization(권한 부여)
- 검증된 사용자에게 어떤 수준(Level)의 권한과 서비스를 허용

Accounting(계정 관리)
- 사용자의 자원에 대한 사용 정보를 모아서 과금, 감사, 용량증설, 리포팅 등에 활용
- 인증받은 사용자가 무슨 동작을 하는 지를 기록 -> AAA 서버로 전송

예시)
U1 = 비사용자
U2 = 일반 사용자
U3 = 관리자

Authentication 단계
> U1은 비사용자 이므로 인증을 수행할 수 없음 -> 접근 거부
> U2는 일반 사용자 이므로 회원 인증 수단(예를들면, ID + PW)을 통해 일반 사용자임을 증명할 수 있음
> U3는 관리자 이므로 관리자 인증 수단(ID + PW, 또는 이외의 방식)을 통해 관리자임을 증명할 수 있음

Authorization 단계
> U1은 접근 거부됨
> U2는 일반 사용자 이므로 관리자의 기능을 제외한 제한된 사용 권한등이 부여됨
> U3는 관리자 이므로 모든 권한을 사용할 수 있음

Accounting 단계
> U1은 접근 거부됨
> U2는 일반 사용자 이므로 사용 기록등이 남게됨
> U3는 관리자 이므로 모든 권한에 대한 사용 기록이 남고, 기록에 대해 접근을 할 수 있음
