※ PPP 인증 방식

PPP(Point-to-Point Protocol)

네트워크 분야에서 두 통신 노드 간의 직접적인 연결을 위해 일반적으로 사용되는 데이터 링크 프로토콜이다.
점대점 프로토콜은 인증, 암호화를 통한 전송 및 데이터 압축 기능을 제공한다.

점대점 프로토콜은 시리얼 케이블, 전화선, 트렁크 라인, 이동 통신망, 특별한 통신 링크(specialized radio link) 및 SONET과 같은 광통신망에서 사용 가능하다.
대부분의 ISP(Internet Service Provider)는 고객이 인터넷에서 접속하기 위한 다이얼 업 접속으로 PPP를 사용하도록 하고있다.
PPP는 크게 PPPoE(Point-to-Point Protocol over Ethernet) 와 PPPoA(Point-to-Point Protocol over ATM) 로 나뉘며,
ISP는 DSL(Digital Subscriber Line) 인터넷 서비스를 위해 이 프로토콜을 사용한다.

PPP는 일반적으로 데이터 링크 계층(장치 간 신호를 전달하는 물리 계층을 이용하여 네트워크 상의 주변 장치들 간 데이터를 전송한다)
또는 다양한 네트워크 계층(중간 라우터를 통한 라우팅을 포함하여 패킷 포워딩을 담당한다)에서 사용할 수 있도록 고안되었다.
브로드밴드(Broadband 광대역 : 하나의 전송 매체에 여러 개의 데이터 채널을 제공한다) 연결에도 사용할 수 있다.

※ 인증 방법

PAP(Password Authentication Protocol)

Plain Text(평문)을 사용하여 인증한다.
인증 절차가 2-Way-Handshake로 동작한다.

CHAP(Challenge Handshakes Authentication Protocol)

Hash(MD5)방식을 사용하여 인증한다.
인증 절차가 3-Way-Handshake로 동작한다.

PPP는 L2 Protocol이기 때문에 PPP인증이 실패하는 경우 Line 프로토콜 상태가 Down이 되기 때문에 해당 회선으로는 통신이 불가능하다.
